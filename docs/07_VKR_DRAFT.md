# ВКР — каркас пояснительной записки (черновик)

**Версия:** v1.8 11.02.2026

**Назначение:** структура пояснительной записки и привязка материалов проекта (SQLite, отчёты, UI, испытания) к разделам ВКР.

## 0. Откуда брать материал
- Постановка/требования: `01_CANON.md`
- Архитектура + данные + карта кода: `02_ARCHITECTURE_AND_DATA.md`
- UI-канон: `03_UI_CANON.md`
- Термины: `04_GLOSSARY.md`
- Испытания + трассируемость: `05_TEST_PLAN_AND_TRACEABILITY.md`
- Журнал/guardrails: `06_CODEX_RULES_AND_CHANGELOG.md`

## 1. Рекомендуемая структура ВКР
### 1. Введение
- актуальность (эксплуатационная диагностика; воспроизводимость результатов)
- цель и задачи
- объект/предмет
- ограничения безопасности и законности тестирования

### 2. Аналитическая часть
- обзор аналогов (UI/HTTP/Network tools), сравнение по критериям
- формирование требований (инварианты из Канона)
- обоснование технологий: .NET 8, Avalonia 11, Playwright, SQLite

### 3. Проектная часть
- архитектура (слои, зависимости, контракты)
- жизненный цикл прогона (Validate → Preflight → Execute → Persist → Report)
- модель данных (SQLite таблицы и связи)
- модель артефактов (`runs/{RunId}/`)
- форматы отчёта (JSON обязательно, HTML опционально)

### 4. Реализация
- ключевые классы/файлы (точки входа)
- реализация 10 модулей (по семействам) — честный MVP
- хранение конфигов/версий (TestCases/TestCaseVersions)
- хранение прогонов/результатов (TestRuns/TestRunItems/Artifacts)

### 5. Испытания
- smoke и демонстрационные сценарии по модулям
- анализ соответствия требованиям (таблица трассируемости)

### 6. Заключение
- достигнутые результаты
- перспективы развития (stress/soak, snapshot-diff, расширенные baseline-проверки)

## 2. Что важно показать по требованию кафедры/стандарта
- использование БД **по делу**: конфиги/версии/профили/прогоны/артефакты.
- отчётность: пример `report.json` (и `report.html`, если включено).
- демонстрация работоспособности системы на ПК.

> Примечание: правила оформления (методичка/стандарт предприятия) применяются на этапе финальной верстки текста; этот файл держит структуру и связность материалов.

## 3. Приложения (что приложить к ВКР)
- примеры `report.json`/`report.html`
- скриншоты основных экранов (4 вкладки + лог drawer)
- листинги ключевых классов (оркестратор, store, artifact store)
- матрица трассируемости (см. `05_...`)
