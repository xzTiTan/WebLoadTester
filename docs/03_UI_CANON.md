# 03 — UI канон (якорный документ)

**Версия:** v3.39 23.02.2026
**Статус:** TO-BE детализация для `docs/INDEX.md`.

> Источник истины: `docs/INDEX.md`. Этот файл фиксирует практические правила UI/UX без изменения продуктовых контрактов.

## 1. Верхний уровень
- 4 вкладки: «UI тестирование», «HTTP тестирование», «Сеть и безопасность», «Прогоны».
- Лог-дровер всегда доступен и отображает диагностику выполнения.
- Глобальные статусы (БД/Telegram/Playwright) показываются в шапке и не блокируют остальные вкладки.

## 2. ModuleWorkspace
- Компоновка: `LeftNav / Content / Details`.
- В центральной форме: один основной вертикальный `ScrollViewer` на рабочую область.
- Должны соблюдаться anti-overlap правила:
  - без наложений при минимальном размере окна,
  - длинные строки не ломают layout,
  - таблицы/карточки остаются доступными через скролл.
- Переключение модулей во время Running блокируется с понятной причиной в логе/индикаторе.

## 3. Состояния и блокировки
- Running state:
  - блокирует действия, нарушающие консистентность (смена модуля/конфигурации в недопустимых точках),
  - не должен «залипать» после ошибок/отмены/stop.
- Stop/Cancel semantics:
  - Stop — мягкая остановка (не стартовать новые итерации),
  - Cancel — немедленная отмена через `CancellationToken`.

## 4. Правила валидации и сообщений
- Ошибки показываются inline + агрегировано (без спама).
- Тексты ошибок — user-friendly и на русском.
- Старт/Сохранение блокируются только при реальных validation errors.

## 5. Runs tab
- Содержит фильтры, таблицу прогонов, панель деталей и действия с артефактами.
- Repeat-run:
  - источник №1 — `report.json` (`moduleId`, `profile`, `moduleSettings`),
  - fallback — DB snapshot,
  - автозапуск не выполняется.
