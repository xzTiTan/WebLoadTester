# Глоссарий — WebLoadTester

**Версия:** v1.1 11.02.2026

Единые определения терминов для UI/кода/ВКР.

- **Модуль (Module)** — функциональный тип теста (A1–C4), реализует `ITestModule`.
- **Семейство (Family)** — группа модулей: UI / HTTP / NET.
- **Конфигурация модуля (TestCase)** — именованный набор настроек модуля (логический «тест»).
- **Версия конфигурации (TestCaseVersion)** — конкретная версия настроек `TestCase` (payloadJson), создаётся при каждом сохранении.
- **Профиль запуска (RunProfile)** — общий набор параметров запуска (mode/iterations/duration/parallelism/timeouts + флаги).
- **Snapshot профиля (ProfileSnapshotJson)** — зафиксированная копия профиля, сохранённая в записи прогона и отчёте.
- **Прогон (Run/TestRun)** — фактический запуск модуля с уникальным `RunId`.
- **RunId** — идентификатор прогона (строка), используется как ключ в SQLite и как имя папки в `runs/`.
- **Артефакт (Artifact)** — файл, созданный в ходе прогона: `report.json`, `report.html`, `run.log`, screenshots и др.
- **report.json** — обязательный машинно-обрабатываемый отчёт.
- **report.html** — опциональный человекочитаемый отчёт.
- **Log Drawer** — общий нижний выдвижной блок лога (скрыт по умолчанию).
- **Preflight** — быстрые проверки готовности (C4), выполняются перед основным модулем по флагу профиля.
- **Baseline (Security baseline)** — безопасные проверки конфигурации/заголовков/минимального TLS без атак.
