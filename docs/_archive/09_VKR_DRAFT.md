
# ВКР — единый текст (черновик структуры)

**Версия:** v1.7 03.02.2026

**Назначение:** каркас пояснительной записки, собирает материалы из якорных документов.  
**См. также:** [01_CANON.md](01_CANON.md), [02_TECH_SPEC.md](02_TECH_SPEC.md), [03_UI_SPEC.md](03_UI_SPEC.md), [04_CODE_MAP.md](04_CODE_MAP.md), [06_TEST_PLAN.md](06_TEST_PLAN.md), [07_TRACEABILITY.md](07_TRACEABILITY.md).

## 0. Как работать с этим файлом
Этот документ — *каркас* пояснительной записки ВКР. 
Содержательное наполнение берётся из якорных файлов:
- требования и постановка: `01_CANON.md`
- архитектура/данные/отчёты: `02_TECH_SPEC.md`
- UI: `03_UI_SPEC.md`
- реализация (код): `04_CODE_MAP.md`
- испытания: `06_TEST_PLAN.md`
- трассируемость: `07_TRACEABILITY.md`

## 1. Введение
1.1 Актуальность (контекст заказчика, ручные проверки → проблемы воспроизводимости)
1.2 Цель и задачи работы
1.3 Объект и предмет исследования
1.4 Требования и ограничения (безопасность и законность тестирования)

## 2. Аналитическая часть
2.1 Обзор аналогов (инструменты UI/HTTP/диагностики) — сравнение по критериям
2.2 Выводы по анализу и формирование требований
2.3 Обоснование выбора технологий (.NET 8, Avalonia, Playwright, SQLite)

## 3. Проектная часть
3.1 Архитектура системы (слои, контракты, жизненный цикл прогона)
3.2 Модульная структура (3 семейства / 10 модулей)
3.3 Модель данных (SQLite) и модель артефактов (runs/<runId>/)
3.4 Форматы отчётов (report.json всегда; report.html опционально)
3.5 UI-архитектура (4 вкладки, единый паттерн экрана)

## 4. Реализация
4.1 Реализация ядра и оркестратора
4.2 Реализация модулей (по семействам)
4.3 Реализация хранения (SQLite) и артефактов
4.4 Реализация отчётов JSON/HTML
4.5 Интеграция Telegram (опционально)

## 5. Испытания и оценка результата
5.1 План испытаний
5.2 Демонстрационные сценарии
5.3 Анализ результатов и соответствие требованиям

## 6. Заключение
6.1 Итоги и достигнутые результаты
6.2 Перспективы развития (stress/soak, snapshot diff, расширенные проверки)
