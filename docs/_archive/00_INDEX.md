# Индекс документации WebLoadTester

**Версия:** v2.9 15.02.2026

## Назначение
Этот файл — единая точка входа в документацию проекта. Используйте его для быстрого выбора нужного документа под задачу: от канона требований до тест-плана и материалов ВКР.

## Карта файлов `docs/`

1. [01_CANON.md](01_CANON.md)  
   **Что это:** канон проекта: требования, инварианты, границы MVP, критерии приёмки.  
   **Когда читать:** всегда первым при изменении поведения, модулей, инвариантов.

2. [02_ARCHITECTURE_AND_DATA.md](02_ARCHITECTURE_AND_DATA.md)  
   **Что это:** архитектура слоёв (Core/Infrastructure/Modules/Presentation), жизненный цикл прогона, модель данных SQLite, структура артефактов.  
   **Когда читать:** перед правками в сервисах, контрактах, хранении, оркестрации.

3. [03_UI_CANON.md](03_UI_CANON.md)  
   **Что это:** UI-канон (4 вкладки, layout-инварианты, поведение экранов и UX-ограничения).  
   **Когда читать:** перед правками XAML/Styles/ViewModel UI-потоков.

4. [04_GLOSSARY.md](04_GLOSSARY.md)  
   **Что это:** единые термины и определения проекта.  
   **Когда читать:** при подготовке текстов, отчётов, ВКР и унификации формулировок.

5. [05_TEST_PLAN_AND_TRACEABILITY.md](05_TEST_PLAN_AND_TRACEABILITY.md)  
   **Что это:** тест-план, smoke-сценарии и таблица трассируемости «требование → реализация → проверка».  
   **Когда читать:** перед приемкой изменений и проверкой покрытия требований.

6. [06_CODEX_RULES_AND_CHANGELOG.md](06_CODEX_RULES_AND_CHANGELOG.md)  
   **Что это:** guardrails для Codex + журнал изменений.  
   **Когда читать:** перед внесением правок и при фиксации результата изменений.

7. [07_VKR_DRAFT.md](07_VKR_DRAFT.md)  
   **Что это:** каркас пояснительной записки ВКР и привязка артефактов проекта к разделам ВКР.  
   **Когда читать:** при подготовке академических материалов.

8. [08_FULL_SOFTWARE_DESCRIPTION.md](08_FULL_SOFTWARE_DESCRIPTION.md)  
   **Что это:** единое цельное описание ПО: назначение, архитектура, пайплайн выполнения, модули, UI и ограничения MVP.  
   **Когда читать:** для быстрого понимания системы целиком (комиссия, новый участник, ревью).

## Рекомендуемый порядок чтения
1. **Канон:** `01_CANON.md`.
2. **Архитектура и данные:** `02_ARCHITECTURE_AND_DATA.md`.
3. **UI-канон:** `03_UI_CANON.md`.
4. **Тест-план и трассируемость:** `05_TEST_PLAN_AND_TRACEABILITY.md`.
5. **Правила Codex и changelog:** `06_CODEX_RULES_AND_CHANGELOG.md`.
6. **Черновик ВКР:** `07_VKR_DRAFT.md`.
7. **Полное сводное описание:** `08_FULL_SOFTWARE_DESCRIPTION.md` (как интегральный документ для защиты/обзора).
