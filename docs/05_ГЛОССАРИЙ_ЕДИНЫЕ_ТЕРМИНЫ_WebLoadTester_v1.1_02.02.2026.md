
# Глоссарий и единые термины — WebLoadTester

**Версия:** v1.1 02.02.2026

## 1. Базовые сущности
- **RunId** — уникальный идентификатор прогона (строка GUID без разделителей), используется как ключ для папки артефактов и связки с БД.
- **TestCase** — тест как сущность: имя + тип модуля + метаданные.
- **TestCaseVersion** — версия теста (payload настроек/сценария). Каждое сохранение создаёт новую версию.
- **RunProfile** — профиль запуска (общие параметры: режим, параллельность, таймауты, флаги).
- **ProfileSnapshot** — зафиксированное значение профиля на момент запуска, хранится в `TestRuns.profileJson` и в `report.json`.
- **TestRun** — запись о выполненном прогоне.
- **TestRunItem** — элемент результата прогона (check/run/probe/timing).
- **Artifact** — файл, относящийся к прогону (report/log/screenshot).

## 2. Термины режимов
- **Iterations** — режим выполнения фиксированного числа итераций.
- **Duration** — режим выполнения в течение фиксированного времени.
- **Stress/Soak/Endurance** — *не входят в ядро*, перечисляются только как перспектива.

## 3. Семейства модулей
- **UI тестирование** — модули на основе Playwright (интеракции и измерения UI).
- **HTTP тестирование** — модули на основе HttpClient (endpoint’ы/ассеты/метрики).
- **Сеть и безопасность** — модули диагностики DNS/TCP/TLS, доступности и базовой безопасности.

## 4. Результаты
- **ModuleResult** — результат модуля (статус + список ResultBase + артефакты).
- **TestReport** — унифицированный отчёт, из которого формируются JSON/HTML.
- **Success/Failed** — успех/провал отдельного пункта результата.

## 5. Политики
- **ScreenshotsPolicy** — политика скриншотов (Never/OnError/Always).
- **TelegramEnabled** — флаг профиля запуска: включать ли уведомления.
