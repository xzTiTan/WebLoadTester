# Codex: шаблоны промптов и журнал изменений — WebLoadTester

**Версия:** v1.0 02.02.2026

## 1. Правило работы с Codex (коротко)
- Каждый запрос Codex = **одна цель** + **границы** + **критерий готовности**.
- Всегда требуем: *не ломать публичные контракты*, *обновить связанные места*, *не допустить UI-наложений*.

---

## 2. Шаблон промпта: исправление ошибки сборки/рантайма

**Контекст:**  
- Ветка: `master` (или другая)  
- Ошибка/stacktrace: (вставь)  
- Файлы/строки: (вставь)  

**Задача:** исправить ошибку и обновить связанные места, чтобы проект компилировался и не появились новые предупреждения.

**Ограничения:**
- Не менять архитектурные якоря: 10 модулей, 3 семейства, 4 вкладки, JSON всегда.
- Не добавлять JavaScript.
- UI не должен иметь наложений; использовать ScrollViewer по спецификации UI.

**Готово когда:**
- Ошибка устранена.
- Добавлен/обновлён минимальный тест/проверка (если уместно).
- Обновлены 1–2 места “по цепочке” (например: VM + View + ресурсы).

---

## 3. Шаблон промпта: новая фича в модуле

**Задача:** добавить (описание фичи) в модуль (A1..C4).  
**Данные:** какие поля в UI, какой JSON будет храниться в PayloadJson, какая запись пойдёт в RunItems/Artifacts.

**Готово когда:**
- UI: добавлены поля/кнопки, валидация, safe defaults.
- Storage: versioning TestCaseVersion работает.
- Reporting: report.json содержит новые поля/метрики.
