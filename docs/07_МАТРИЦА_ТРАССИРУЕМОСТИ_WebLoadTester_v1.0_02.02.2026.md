# Матрица трассируемости — требование ↔ документ ↔ код ↔ проверка (WebLoadTester)

**Версия:** v1.0 02.02.2026

## Как заполнять
- **R-XXX** — идентификатор требования.
- “Документ” — ссылка на раздел “Канон проекта…” или “Тех.спецификация…”.
- “Код” — файл/namespace/класс.
- “Проверка” — запись об испытании + RunId + путь к артефакту.
