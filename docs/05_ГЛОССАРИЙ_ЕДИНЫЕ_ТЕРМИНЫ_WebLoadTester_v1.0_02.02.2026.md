# Глоссарий и единые термины — WebLoadTester

**Версия:** v1.0 02.02.2026

## Правило
Для каждого термина фиксируем:
- **Каноническое русское название** (для ВКР и UI),
- **Короткое определение**,
- **Английское/кодовое имя** (для классов/JSON/БД).

---

| Термин (RU) | Определение | Кодовое имя (EN/Code) |
|---|---|---|
| Тест | Описывает проверку в одном модуле: входные параметры + процедура + ожидаемый результат. Хранится в БД, переиспользуется. | TestCase |
| Версия теста | Снимок настроек теста на момент сохранения. Каждое “Сохранить” создаёт новую версию. | TestCaseVersion |
| Профиль запуска | Шаблон параметров исполнения: параллельность, режим, таймауты, политики отчёта/артефактов/Telegram. | RunProfile |
| Прогон | Один запуск теста с конкретной версией и snapshot профиля. Имеет RunId и статус. | TestRun |
| Артефакт | Файл-след прогона: отчёты, логи, скриншоты и т.п. | Artifact |
| Отчёт JSON | Основной машиночитаемый результат прогона; формируется всегда. | report.json |
| Отчёт HTML | Человеко-ориентированный отчёт для руководителя/заказчика; опционально. | report.html |
| Preflight | Предпроверка готовности перед запуском тестов (логически соответствует модулю C4). | Preflight |
| RunItem | Запись детализации выполнения: шаг, endpoint, asset, netcheck и т.д. | RunItem |
| Safe defaults | Безопасные параметры нагрузки по умолчанию, чтобы исключить случайное “переусиление”. | SafeDefaults |

---

## Синонимы, которые запрещаем (чтобы не плодить смыслы)
- “Запуск” вместо “Прогон” (в тексте ВКР) — используем **прогон**.
- “План выполнения” вместо “Профиль запуска” — используем **профиль запуска**.
- “Отчёт” без уточнения — всегда писать **JSON отчёт** или **HTML отчёт**.
